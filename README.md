# ГПК - Интерактивна карта на препратки

Интерактивна уеб апликация за визуализация на Граждански процесуален кодекс (ГПК) като mind map с препратки между членовете.

## Функционалности

- **Фокусирана визуализация**: Mind map показва търсените членове и техните връзки
- **Преки препратки**: Показва 613 директни препратки между членовете (сини стрелки)
- **Семантични връзки**: Открива 56,282 семантични връзки (пунктирани стрелки при активиране)
- **Търсене**: Пълнотекстово търсене по номер на член, заглавие и съдържание
- **Детайли за членове**: Панел с пълната информация за избран член
- **Навигация**: Кликване на препратки за преход между членове
- **Responsive дизайн**: Работи на десктоп и мобилни устройства

## Технологии

- **React 18** - UI библиотека
- **Vite** - Build tool и dev server
- **React Flow** - Mind map визуализация
- **Radial Layout** - Бързо радиално подреждане на графа
- **Python 3** - Парсинг на законовия текст

## Инсталация

1. Инсталирайте зависимости:

```bash
npm install
```

2. Стартирайте development server:

```bash
npm run dev
```

Апликацията ще се отвори на http://localhost:3000

## Използване

### Работен процес

1. **Търсене**:
   - Въведете текст в полето за търсене (номер, заглавие или съдържание)
   - Резултатите се показват в реално време
   - Кликнете на резултат за визуализация

2. **Визуализация**:
   - Mind map показва избрания член и всички свързани членове
   - Сини стрелки = преки препратки
   - Пунктирани стрелки = семантични връзки (при активиране)

3. **Навигация**:
   - Кликнете на член за показване на детайлите му
   - Следвайте препратки в текста за преход към други членове
   - Използвайте zoom/pan контролите за навигация

### Семантични връзки

- Активирайте бутона "Покажи семантични връзки" за показване на свързани членове
- Семантичните връзки са показани с пунктирани линии

## Актуализиране на данните

За да актуализирате данните с нова версия на ГПК:

1. Обновете файла `gpk_text.txt` с новия текст
2. Стартирайте парсъра:

```bash
npm run parse
```

Това ще създаде нов `src/data/gpk_data.json` файл с актуализираните данни.

## Build за продукция

```bash
npm run build
```

Това ще създаде оптимизирана версия в директорията `dist/`.
